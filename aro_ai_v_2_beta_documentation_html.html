ARO AI v2 Beta — Documentation

A concise guide to ARO AI v2 Beta — features, architecture, and roadmap.

Overview

ARO AI focuses on fast, safe, and content-optimized responses. Messages undergo a brief safety review; responses are not streamed but are specially optimized for your content.

Cloudflare-hosted Privacy-first Serverless

Core Technology

  • Custom LLM backbone: Self-trained & fine-tuned models instead of premade APIs (e.g., GPT-4o).
  • Base model: Meta LLaMA 3.2 70B Indestruct — extended & optimized (~100B+ effective parameters).
  • Data storage: No conventional DBs. Packet-based metrics for response times, errors, and pings only.
  • Privacy: Conversations are not stored.
  • Deployment: Serverless on Cloudflare with a dedicated Lenovo IdeaPad x360 server running 24/7.

Performance Improvements

Previous: 1,000 requests/day • ~10 visitors/day

Now: 100,000 requests/day • 200,000+ visitors/day

New Features in ARO AI v2

Personality Mode

Tracks emotional tone and uses a custom emotion database. When enabled, it prioritizes empathetic responses by temporarily overriding certain functions to focus on problem solving and user support.

Web Weaver Mode

Generates professional HTML, CSS, and JS code with frontend preview capabilities. Previews are frontend-only, so some external links or features may not work. v2.5 will add React / Next.js support via WebContainers.

Planned & Upcoming

ARO AI API

Status: In progress. Estimated launch: September (may be delayed). Access will be subscription-based; API logs will record usage metrics.

ARO Offline

Standalone offline models for local use. Targeted model sizes: 0.6B, 1B, 3B, 4B, 7B, 15B, 20B, 25B, 30B. The 0.6B model is ready for production; others are in training.

Image Generation

Not planned for v2.
